# BG-Tate - Форум за бащи

BG-Tate е платформа за бащи, които искат да споделят опит, да получат подкрепа и да си помагат взаимно. Тук можеш да говориш свободно за отглеждането на децата, за трудните моменти в живота и за всичко, за което понякога е трудно да попиташ.

## Функционалности

### Публична част
- Начална страница - Представяне на платформата
- Каталог - Списък с всички теми
- Детайли - Преглед на конкретна тема
- Вход - Форма за вход
- Регистрация - Форма за регистрация

### Частна част (за логнати потребители)
- Създаване на тема - Създаване на нова тема за дискусия
- Редактиране на тема - Редактиране на собствени теми
- Изтриване на тема - Изтриване на собствени теми

## Автентикация

Проектът използва Context API за управление на автентикация:
- Login - Вход с email и парола
- Register - Регистрация на нов потребител
- Logout - Изход от системата
- Route Guards - Защита на маршрутите

## CRUD операции

### Теми (Topics)
- Create - Създаване на нова тема
- Read - Преглед на всички теми и детайли
- Update - Редактиране на собствена тема
- Delete - Изтриване на собствена тема

## Архитектура

Проектът следва структурирана организация на кода:

### Структура на папките

- **src/components/** - Преизползваеми UI компоненти (Button, Card, Container, Navigation, PageTitle, FormField и др.)
- **src/pages/** - Страници за маршрути (Catalog, Details, CreateTopic, EditTopic, Login, Register)
- **src/contexts/** - React Context за управление на глобално състояние (AuthContext за автентикация)
- **src/services/** - API слой за комуникация с backend (api.js, topicService.js)
- **src/guards/** - Route guards за защита на маршрутите (PrivateGuard, GuestGuard)
- **src/utils/** - Помощни функции

### Автентикация

**AuthContext** управлява състоянието на автентикацията, като съхранява данните за потребителя (email и accessToken) в localStorage. Това позволява запазване на сесията след презареждане на страницата.

### Комуникация с API

**topicService** осигурява CRUD операции върху теми чрез комуникация с MockAPI backend. Всички операции изискват accessToken за автентицирани заявки.

### Защита на маршрутите

**Route guards** (PrivateGuard, GuestGuard) защитават страниците според автентикационния статус на потребителя:
- PrivateGuard - достъп само за логнати потребители
- GuestGuard - достъп само за нелогнати потребители

### Права за достъп

Само авторът на темата (потребителят, чийто email съвпада с `topic.ownerId`) може да редактира или изтрива собствената си тема. Другите логнати потребители нямат достъп до тези действия.

### Взаимодействие с записи

Логнатите потребители могат да взаимодействат с темите чрез функционалността за "Харесване". Всеки потребител може да хареса тема веднъж, като броят на харесванията се показва на страницата с детайли.
